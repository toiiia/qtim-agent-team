# /qtim:team-down — свернуть команду

> Читает только team-lead. Канон рантайма — в [`team-up.md`](team-up.md).

## Что делает

В рантайме Agent Teams **нет состояния на диске, которое нужно удалять**: команда существует неявно на время сессии, примитивы `TeamCreate`/`TeamDelete` упразднены. «Свернуть» = остановить фоновые роли и закрыть task list.

**Перед сворачиванием:** для завершённого эпика режима C/D сначала прогони [`/qtim:team-retro`](team-retro.md) — пока роли живы и могут сами записать уроки в свою память (после `TaskStop` им уже не написать).

## Шаги

1. **Останови фоновые роли.** Для каждого члена, поднятого через `Agent({ run_in_background: true })`, вызови `TaskStop` по соответствующему фоновому таску (узнать активные — `TaskList`). `TeamDelete` не используется.
2. **Закрой/зафиксируй task list.** Незавершённые задачи либо доведи, либо отметь статус честно (`TaskUpdate`) — не оставляй «висящих» in_progress без владельца.
3. **Зафиксируй ценное на диске.** Контекст субагентов не переживает сессию — всё, что решено в этой сессии, должно быть в файлах (charter / `memory/*` / персистентная agent-memory ролей), а не «в головах».
4. **Эпик не завершён → запиши `memory/epic-state.md`** (его читает `/qtim:team-up` в новой сессии и предлагает продолжить):

   ```markdown
   # Epic state: <название эпика>
   Обновлено: <дата> · Фаза: design | impl | test | review
   ## Сделано
   ## В полёте (задача — owner — статус — следующий шаг)
   ## Открытые вопросы / блокеры
   ## Следующий шаг при продолжении
   ```

   Эпик завершён — **удали** устаревший `epic-state.md`, не оставляй ложного «в полёте».
5. **Сообщи пользователю** сводку: какие роли остановлены, состояние task list, что зафиксировано.

## Важно

- Liveness привязан к сессии: после рестарта CLI in-process агенты не переживут — `team-down` нужен только в рамках текущей живой сессии.
- Не пытайся `SendMessage` остановленным ролям — их контекст завершён.
