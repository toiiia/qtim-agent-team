# /qtim:onboard — глубокий онбординг памяти

> Читает только team-lead. Setup делает быстрый discovery (5-10 tool calls) — этого хватает для
> charter, но `memory/` рождается тощей. Onboard — отдельный глубокий проход, чтобы команда
> не была «слепой» первые эпики. Когда: сразу после `/qtim:setup` на существующей кодовой базе
> или позже, когда память ощутимо отстаёт от кода. Для greenfield-проекта не нужен.

## Шаги

1. **Инвентаризация.** Прочитай charter (если есть) и текущее `memory/` — что уже записано,
   дубликаты не создавать. Составь список подсистем для исследования: слои из discovery
   (фронт-модули, серверные routes, схема БД и миграции, realtime, инфра/CI) + крупные каталоги.
2. **План и подтверждение объёма.** Покажи список исследователей (подсистема → что ищем)
   и объём (N параллельных агентов). Fan-out жжёт токены — дождись ok пользователя.
3. **Fan-out (режим C — параллельные `Agent`, НЕ Workflow).** На подсистему — `Explore` /
   `general-purpose` с чётким заданием:
   - точки входа и карта модуля;
   - реальные конвенции (именование, паттерны) с прецедентами `file:line`;
   - инварианты, **соблюдаемые кодом** (не желаемые): владение данными, каскады, проверки
     доступа, переходы состояний;
   - известные грабли (TODO/FIXME/workaround-комментарии, подозрительные обходы).
   Исследователь возвращает структурированный ВЫВОД — в память сам не пишет.
4. **Синтез (team-lead).** Сведи отчёты в файлы `memory/` по структуре из charter (карта
   архитектуры / инварианты / commands / реестр фич). Конфликт между отчётами — проверь сам
   по коду. Каждый инвариант — с прецедентом `file:line`; гипотезы помечай как гипотезы.
5. **Индекс и отчёт.** Обнови `MEMORY.md`; пользователю — что записано и что осталось белым
   пятном (кандидаты на следующий onboard-проход).

## Правила

- Исследователи read-only: код не правят, в память не пишут — пишет только team-lead
  (контроль дублей и качества).
- Не дублировать CLAUDE.md/README — память дополняет их, не пересказывает.
- Инвариант без подтверждения в коде инвариантом не записывается.
- Объём: файлы памяти — сжатые факты, не эссе; глубокие детали — отдельными topic-файлами
  со ссылкой из `MEMORY.md`.
- Очень большой репозиторий — итерациями: ядро сейчас, периферия следующим проходом.
