# qtim — команда AI-субагентов под твой проект

Плагин для Claude Code, который **одной командой разворачивает в твоём проекте команду специализированных субагентов** — архитектор, разработчики, тестировщик, ревьюер и другие. Вместо одного ассистента-универсала ты получаешь команду с чётким разделением ролей, общим списком задач и контролем качества, которая ведёт задачу от проектирования до финального ревью.

Плагин подстраивается под стек: анализирует проект, задаёт несколько вопросов и генерирует роли под твои фреймворки.

## Что это даёт

- **Разделение труда** — каждая роль отвечает за свой слой: БД, фронтенд, тесты, ревью, архитектура. Меньше «каши» в одной голове.
- **Подстройка под стек** — команда генерируется под твой проект (Nuxt/Next/NestJS/Postgres/…), а не по generic-шаблону.
- **Автоподбор инструментов** — при настройке плагин предлагает skills и расширения, релевантные твоему стеку.
- **Качество из коробки** — встроенные гейты: тесты, код-ревью, проверка результата перед завершением.
- **Гибкие режимы** — от лёгкого (одна задача) до полного эпик-флоу с циклами «сделал → проверили → доработал».

## Кто в команде

До 9 ролей (генератор подберёт нужные под твой проект):

- **architect** — проектирует, пишет ADR, следит за границами модулей
- **db / front** — разработчики своих слоёв (миграции и API / UI и компоненты)
- **tester** — реальные прогоны в браузере, не только проверки в коде
- **reviewer** — финальный гейт качества (безопасность, производительность)
- **explorer** — быстрый поиск по репозиторию
- **devops / product / auditor** — инфраструктура, UX и проактивный аудит

## Требования

Нужен **Claude Code с включённым Agent Teams** — это мультиагентный режим, на котором работает команда. Включается флагом в `settings.json` (глобально `~/.claude/settings.json` или в проекте `.claude/settings.json`):

```json
{ "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" } }
```

## Установка

```bash
/plugin marketplace add toiiia/qtim-agent-team
/plugin install qtim@qtim-agent-team
```

## Быстрый старт

1. Открой Claude Code в корне своего проекта.
2. Разверни команду под него:
   ```
   /qtim:setup
   ```
   Плагин разберёт стек, задаст несколько вопросов и создаст команду прямо в проекте.
3. Работай через команды:
   ```
   /qtim:team-up      # поднять полную команду — для крупной задачи
   /qtim:team-lazy    # лёгкий режим — роли по мере надобности
   /qtim:team-down    # свернуть команду
   ```

## Команды

| Команда | Когда использовать |
|---|---|
| `/qtim:setup` | Один раз в новом проекте — развернуть команду под стек |
| `/qtim:team-up` | Крупная задача/эпик с циклами «сделал → проверили → доработал» |
| `/qtim:team-lazy` | Быстрая или средняя задача без полного прогрева |
| `/qtim:team-down` | Остановить команду (незавершённый эпик сохранится и продолжится в новой сессии) |
| `/qtim:team-sync` | После обновления плагина — подтянуть изменения в собранную команду |
| `/qtim:team-retro` | После эпика — дистилляция уроков в память команды |
| `/qtim:onboard` | Глубокое наполнение памяти на существующей кодовой базе |
| `/qtim:doctor` | Диагностика: флаг Agent Teams, charter, агенты, настройки, codex |

## Обновление

Плагин обновляется через маркетплейс (`/plugin` → Manage marketplaces → update, или `claude plugin update qtim@qtim-agent-team`). После обновления зайди в проект с собранной командой и запусти `/qtim:team-sync` — он перенесёт изменения движка в сгенерированные файлы (charter, роли, настройки), не трогая твою проектную конкретику и память. Если команда собрана по старой версии, плагин сам подскажет об этом при старте сессии.

## Как это выглядит

```
Ты:       /qtim:team-up, затем — «добавь раздел избранного: БД, API и UI»
Команда:  architect проектирует → db делает миграцию → front пишет UI →
          tester прогоняет в браузере → reviewer даёт вердикт
Ты:       получаешь готовый результат, а не сырой процесс
```

## Что появится в проекте

После `/qtim:setup` в твоём проекте создаются описание команды (`team-charter.md`), роли-субагенты под твой стек и базовая память. Сам движок команд живёт в плагине — проект остаётся чистым.
