# Examples — golden-референс генератора

`nuxt-supabase/` — эталонный вывод `/qtim:setup` для вымышленного проекта **acme**
(Nuxt 3 + Supabase, состав Standard, intake «автопилот с асимметрией», Codex Yes,
форма Plugin-linked). Показан репрезентативный срез: charter целиком, одна сгенерированная
роль (`db` — самая насыщенная подстановками), settings и baseline-память.

Назначение:

1. **Живая документация** — что именно появится в проекте после `/qtim:setup`.
2. **Golden-тест генератора** — правишь шаблоны `agents/` или структуру charter в setup
   Phase 4.1 → обнови эталон и сверь дифф глазами: неожиданные изменения в эталоне =
   регрессия генерации. CI проверяет, что в эталоне не осталось плейсхолдеров `{{...}}`.

Это НЕ рабочий проект: кода нет, пути и значения вымышленные, абсолютный путь к
codex-протоколу — иллюстративный (реальный резолвит setup при генерации).
